================================================================================
ARCHITECTURE MOCK
================================================================================

Les tests n'utilisent PAS de vraie base de données :

├─ jest.mock('../../src/lib/prisma')
│  └─ Retourne mockPrisma avec toutes les méthodes mockées
│
├─ Chaque test fait :
│  ├─ jest.clearAllMocks() dans beforeEach()
│  ├─ Configure les mocks selon le test (prismaMock.*.mockResolvedValue(...))
│  └─ Vérifie que les appels attendus ont été faits
│
└─ Routes testées avec :
   ├─ express() app locale (pas d'import server.js)
   ├─ supertest(app).get/post(url) pour les requêtes HTTP
   └─ Mocks des dépendances (Prisma, config, jwt, etc.)

================================================================================
CONVENTIONS
================================================================================

✅ Setup
  └─ beforeEach(() => jest.clearAllMocks())

✅ Nommage
  ├─ Describe : "Entité ou Fonction"
  ├─ It : "should do X when Y"
  └─ Tests groupés par endpoint/fonction

✅ Mocks Prisma
  ├─ jest.mock('../../src/lib/prisma', () => require('../helpers/mockPrisma'))
  ├─ const { prismaMock } = require('../helpers/mockPrisma')
  ├─ prismaMock.*.mockResolvedValue(data)  // Async
  └─ prismaMock.*.mockReturnValue(data)    // Sync

✅ Routes avec supertest
  ├─ const request = require('supertest')
  ├─ const app = express()
  ├─ const res = await request(app).get(url)
  └─ expect(res.status).toBe(200)

✅ JWT dans les tests
  ├─ Créés avec jwt.sign(payload, JWT_SECRET, { expiresIn: '30d' })
  ├─ Passés en Authorization: Bearer <token>
  └─ Vérifiables avec jwt.verify(token, JWT_SECRET)

================================================================================
POINTS IMPORTANTS À NOTER
================================================================================

Routes search.js sans await
  Les routes /api/search, /api/producer, /api/region appelent les fonctions
  async WITHOUT await → retournent des Promises immédiatement.
  Les tests mockent les fonctions pour retourner des valeurs synchrones.

DATABASE_URL non requis
  Les tests passent sans DATABASE_URL configurée car Prisma est mocké.
  La vraie DB n'est JAMAIS contactée.

Ordre des jest.mock()
  Les jest.mock() DOIVENT venir avant les require() du module testé.
  Sinon les imports réels seront utilisés à la place du mock.

Tokens JWT à chaque test
  Créés dynamiquement dans beforeEach ou avant chaque test qui en a besoin.
  Permet de tester expiration, invalidité, etc. sans duplication.
