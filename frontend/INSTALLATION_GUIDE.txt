ÉTAPE 6 : Mode watch (développement)
================================================================================

Pour développer avec rechargement automatique :

npm run test:watch

Cela va :
- Afficher tous les tests
- Écouter les changements de fichiers
- Relancer automatiquement les tests affectés
- Afficher les résultats en temps réel

Commandes dans le mode watch :
  a     : Relancer tous les tests
  f     : Relancer uniquement les tests échoués
  p     : Relancer par pattern (fichier/fonction)
  q     : Quitter le mode watch

Pour quitter : appuyez sur q

================================================================================
TROUBLESHOOTING RAPIDE
================================================================================

PROBLÈME : npm install échoue
SOLUTION :
  npm install --verbose
  Vérifier que Node.js 16+ est installé (node --version)

PROBLÈME : vitest command not found
SOLUTION :
  npm install -D vitest

PROBLÈME : localStorage is not defined
SOLUTION :
  Vérifier que tests/setup.ts existe dans le dossier tests/
  Vérifier que vitest.config.ts pointe vers setupFiles: [./tests/setup.ts]

PROBLÈME : Tests ne trouvent pas src/ imports
SOLUTION :
  Vérifier que les imports utilisent ../../src/ (chemins relatifs)
  Exemple : import { getStoredToken } from ../../src/lib/auth

PROBLÈME : Tests très lents
SOLUTION :
  Vérifier que vitest.config.ts a environment: jsdom (non node)
  Fermer autres processus consommant CPU

PROBLÈME : Cannot find module @testing-library/react
SOLUTION :
  npm install -D @testing-library/react jsdom

Lire TESTS_CHECKLIST.md pour plus de solutions.
